Instructions

Goals

The Theme Builder project is set up with a custom theme for you to edit when running on your local machine. This documentation will walk you through how to edit these custom theme files in your favorite text editor to create your custom theme.

Setup

  1. Install the tooling
    1. If you've never installed the tooling see the getting started developer guide chapter
    2. If you have used our tooling before you need to uninstall and reinstall the CLI each time you upgrade to a new version of JET, here are the commands:
      • npm uninstall -g ojet-cli
      • npm install -g @oracle/ojet-cli
  2. Create a directory named ThemeBuilder (or whatever name you'd like)
  3. In the ThemeBuilder directory unzip JET-Theme-Builder.zip
  4. In your CLI cd into the ThemeBuilder directory
  5. Once in the ThemeBuilder directory run the following setup commands:
    1. ojet restore
    2. ojet add sass
    3. ojet create theme mytheme
  6. Any time you want to start up run the command below from the ThemeBuilder directory (and any time you want to stop it press ctrl-C in the CLI window):
    • ojet serve --themes=all:all

Resources

Tutorial

To familiarize yourself with how to write a custom theme:
  • On the home page under 'Custom Theme' choose the green theme (click the button with the leaf icon ).
  • Now click the 'Files' button.
  • Follow the tutorial to see the files for this theme.

You can repeat the tutorial for other themes like stealth and san francisco. Each demo theme has a different focus, click the help icon next to 'Custom Theme' for more information.

Edit 'My Theme'

The following assumes you are running Theme Builder on your local machine.
  1. Select 'My Theme':
    • go to the Home page and under 'Target Platform' make sure the first button for web is selected. Under 'Custom Theme' select the wrench icon . Under 'View Selection' make sure 'Components' is selected.
  2. Change a variable:
    • in your favorite text editor open [ROOT]/src/themes/mytheme/web/_mytheme.web.settings.scss. Uncomment $interaction1Color and change the value to red. Wait for the css file to be regenerated (the page will reload). You should now see that in many places the dominant color of blue has been replaced with red.
    • Some high impact variables that you can modify to see their effect include:
      • Palette variables: $interaction1Color, $background1Color, and $border1Color
      • Text variables: $fontFamily, $rootFontSize, $textColor, and $linkTextColor
    • More information about variables is available in the 'SCSS Variables' section of the developer guide.

The instructions above explain editing the files in the mytheme 'web' directory, but similar files are available in the sibling directories 'ios', 'android', and 'windows'.

Performance

For performance reasons you only want to download what you really need, see the Controlling CSS Content in the theming developer guide for information about controlling which css is generated.

Theme Builder may become unusable if you choose not to include certain things, but just to see how it works open the custom theme settings file ([ROOT]/src/themes/mytheme/web/_mytheme.web.settings.scss) and uncomment '$includeFormControlClasses' and set the value to false. Wait for the page to reload. Now click on the 'Forms' tab, the form controls should look unstyled because the css is no longer included.

JET Sass Files

You may want to search through JET's scss files, for example to see where a variable is used. JET's scss files can be found in [ROOT]/node_modules/@oracle/oraclejet/dist/scss. However you should never edit the files in these directories as it makes upgrading to a new version of JET difficult. Instead you should only edit files in your [ROOT]/src/themes/ directories.
